Preguntas

  • ¿Cuáles son las diferencias entre los posibles valores de la propiedad position?
    1. Static: Es el valor por defecto. Los elementos no se ven afectados por las propiedades 'top', 'bottom', 'left' y 'right'. Se posiciona de acuerdo al flujo de la página.
    2. Relative: El elemento es posicionado relativamente a su posición normal. Las propiedades 'top', 'bottom', 'left' y 'right' harán que se ajuste diferente de su posición normal.
    3. Fixed: El elemento está posicionado relativamente a la venta del navegador, lo que significa que se quedará en el mismo lugar aunque nos movamos por la página. Las propiedades 'top', 'bottom', 'left' y 'right' harán que se ajuste a esta posición.
    4. Absolute: El elemento se posiciona en relación con ancestro posicionado más cerca, si no hay ancestro lo hace con el cuerpo del documento y se mueve por la página.
    5. Sticky: El elemento se posiciona en base al movimiento del usuario. Se posiciona relativamente hasta que se mueve, luego se pega al lugar donde queda, tal y como un fixed element.
  • ¿Cuáles son los valores estándar para la propiedad visibility?
    1. Visible: Es el valor por defecto. El elemento es visible.
    2. Hidden: El elemento está oculto, pero sigue ocupando espacio.
    3. Collapse: Solo para filas en tablas (tr), grupos de filas (tbody), columnas (col) y grupos de columnas (colgroup). Este valor remueve una fila o columna, pero no afecta el layout de la table, el espacio se pone disponible para otros.
      Si collapse se usa en otros elementos, se comporta igual que hidden.
    4. Initial: Establece la propieda al valor por defecto (se puede personalizar).
    5. Inherit: Hereda la propiedad de su elemento padre.
  • ¿Qué es el zIndex y para qué sirve?
    Es una propiedad que permite especificar las posiciones de diversos elementos con respecto al eje Z, por lo que se pueden sobreponer algunos elementos con respecto a otros; por ejemplo, poner una imágen detrás de texto (al poner 'z-index = -1' en el CSS para img).
