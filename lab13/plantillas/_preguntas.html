¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

session_unset() - Libera todas las variables de la sesión, es decir, libera todas las variablas de sesión que están almacenadas actualmente.

session_destroy() — Destruye toda la información asociada con una sesión actual, más no hace el unset de las variables globales asociadas con la sesión, ni tampoco hace el unset de las cookies de la sesión, de ahí la importancia de usar ambas funciones.

¿Cuál es la diferencia entre una variable de sesión y una cookie?

Una cookie es un bit de información guardada por el navegador y enviada al servidor con cada request.

Una variable de sesión es parte de una colección de información guardada en el servidor, asociada con cierto usuario. A diferencia de una cookie, la información no se guarda en la computadora del usuario.

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Un método bastante útil es asociar el ID del usuario con un número random (utilizando alguna función como uniqid() de PHP), para luego revisar que no se haya repetido, aún siendo random. Se pueden utilizar tablas hash.

¿Qué es CSRF y cómo puede prevenirse?

Cross-Site Request Forgery (CSRF) es un tipo de ataque que forza a un usuario a ejecutar acciones no deseadas on una aplicación web donde están logeados. Los ataques de este tipo apuntan a cambiar el estado de las requests, más no a robar información, ya que el eatacante no puede ver la respuesta al request forzado.

Algunas maneras de prevención son las siguientes:

  1. Usar el método POST y no GET.
  2. Double Submitted Cookie, que es generar un valor aletorio cuando un usuario visita un sitio, y pedir ese valor en cada submission o post request.
  3. Hacer un token con cada forma.
