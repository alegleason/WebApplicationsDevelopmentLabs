Preguntas

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
    Si tu JavaScript se encarga de validar los datos antes de ser enviados obtendrás dos beneficios principales: Primero te estás asegurando que el usuario haya llenado todos los campos con información válida, por lo que el servidor ya no tiene que validarla. Y en segunda, te aseguras que todos los datos que sean pasados a la base de datos estén con el formato adecuado para evitar correciones posteriormente.
  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
    Puedes deshabilitar JavaScript en tu navegador. Todos los navegadores cuentan con la opción de deshabilitar JavaScript.
  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
    Siempre es buena práctica validar las cosas antes de transferirlas al servidor, a pesar de que este igualmente puede validarlas, si la forma es rechazada antes de ser pasada al servidor, mejorará el tiempo de respuesta.
