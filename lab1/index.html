Conceptos

  • URL Según autores de Tecnología & Informática, URL es una dirección única que poseen todos los elementos en Internet. URL son las siglas en inglés de “Uniform Resource Locator”, cuyo significado en español es Localizador Uniforme de Recursos, y que sirve, como se menciona más arriba, para nombrar recursos en Internet. [1]
  • MIME Según colaboradores de Mozilla un MIME es un tipo Extensiones Multipropósito de Correo de Internet (MIME) es una forma estandarizada de indicar la naturaleza y el formato de un documento. Está definido y estandarizado en IETF RFC 6838. Los navegadores a menudo usan el tipo MIME (y no la extensión de archivo) para determinar cómo procesará un documento. Ejemplo: image/gif. [2]
  • Métodos y códigos de errores HTTP Según colaboradores de Mozilla los códigos de estado de respuesta HTTP indican si se ha completado satisfactoriamente una solicitud HTTP específica. Las respuestas se agrupan en cinco clases: respuestas informativas, respuestas satisfactorias, redirecciones, errores de los clientes y errores de los servidores. Los códigos de estado se definen en la sección 10 del section 10 of RFC 2616. Los métodos HTTP permiten comunicar al servidor lo que se quiere realizar con un resource bajo una URL. [3]
  • XML Según colaboradores de Mozilla XML es un lenguaje de marcado similar a HTML. A diferencia de otros lenguajes de marcado, XML no está predefinido, por lo que debes definir tus propias etiquetas. El propósito principal del lenguaje es compartir datos a través de diferentes sistemas, como Internet. [4]
  • XHTML Según colaboradores de Mozilla XML es a XML como HTML es a SGML. Es decir, XHTML es un lenguaje de marcado que es similar al HTML, pero con un sintaxis más estricta. Es básicamente HTML expresado como XML válido. [5]
  • HTML5 Según Juan Diego Gauchat en su libro "El gran libro de HTML5 CSS3 y Javascript" en 2012, HTML5 es un nuevo concepto para la construcción de sitios web y aplicaciones en una era que combina dispositivos móviles, computación en la nube y trabajos en red. [6] Es la quinta edición del lenguaje básico del Internet, sus siglas significan HyperText Markup Language, versión 5.
  • JS Según colaboradores de Mozilla JavaScript (JS) es un lenguaje ligero e interpretado, orientado a objetos con funciones de primera clase, más conocido como el lenguaje de script para páginas web, pero también usado en muchos entornos sin navegador, tales como node.js, Apache CouchDB y Adobe Acrobat. Es un lenguaje script multi-paradigma, basado en prototipos, dinámico, soporta estilos de programación funcional, orientada a objetos e imperativa. [7]
  • php De acuerdo a la misma pagina de PHP, este es un lenguaje de programación, interpretado, diseñado originalmente para la creación de Páginas web dinámicas. Es usado principalmente en interpretación del lado del servidor (server-side scripting) pero actualmente puede ser utilizado desde una interfaz de línea de comandos o en la creación de otros tipos de programas incluyendo aplicaciones con interfaz gráfica. [8]
  • jsp Según colaboradores de Oracle, JSP es similar a PHP, pero usa el lenguaje de programación Java. Para desplegar y correr JavaServer Pages, se requiere un servidor web compatible con contenedores servlet como Apache Tomcat o Jetty. Provee una manera simple y más rapida de crear contenido web dinámico. [9]
  • Ajax Según Victor Rivas en 2012, AJAX se define como una técnica para el desarrollo de páginas (sitios) web que implementan aplicaciones interactivas. Las siglas significan Asynchronous Javascript and XML. [10]

Preguntas

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web? El internet es un sistema que conforma una gran red de comunicación mundial, a través de la cual se mantienen interconectadas millones de personas por medio de diversos servicios que ofrecen la conexión. La web o World Wide Web es un sistema que funciona a través de internet, por el cual se pueden transmitir diversos tipos de datos a través del Protocolo de Transferencia de Hipertextos o HTTP, que son los enlaces de las páginas web. (Akus.net, s.f.) [11]
  2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE? HTTP: Protocolo de la capa de aplicación tiene varios métodos para establecer comunicación. HEAD: Petición para devolver los headers, no del cuerpo, se usa desde consola para ver si hay comunicación con el servidor. GET: Recupera información, se usa por default al ingresar un URL en la barra de navegación, no utilizar al enviar información sensible. POST: Petición para que el recurso en el URI haga algo con la entidad provista con seguridad básica. PUT: Actualiza información del registro, al igual que PATCH. Guardar una entidad en un URI. PATCH: Actualizar campos específicos de una entidad en un URI. DELETE: Petición para eliminar un recurso. (Spring.io, s.f.)[12]
  3. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? Se debe utilizar el método POST, ya que se encarga de recibir y validar la información.
  4. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? Método GET.
  5. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? La solicitud ha tenido éxito. El significado de un éxito varía dependiendo del método HTTP. [3]
  6. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? No, ya que el servidor no pudo encontrar el contenido solicitado, el usuario debió haber insertado alguna dirección existente [3]
  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? El error 500 está clasificado como error del servidor, el desarrollador necesariamente tiene la responsabilidad. El servidor ha encontrado una situación que no sabe como manejarla. [3]
  8. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML4 que en HTML5 estén desaprobados.

    Un atributo desaprobado es aquel que ha sido reemplazado con otros más funcionales o alternativas flexibles (Codehelp.co.uk, s.f.). [13] HTML5 no tiene ninguno de los atributos de presentación que se encontraban en HTML4 ya que sus funciones se manejan mejor con CSS. Algunos atributos de HTML4 ya no están permitidos en HTML5 y se han eliminado por completo. (tutorialspoint, s.f.) [14]

  9. ¿Cuáles son las diferencias principales entre HTML4 y HTML5? Según Segio Luján en 2011, en HTML5 está mejor definido el manejo de errores, introduce nuevos atributos como autofocus, placeholder, tiene mejores características de aplicaciones web y se mejoró la semántica de los elementos. HTML5 define una sintaxis compatible con HTML4 y XHTML 1.0, por lo que hay varios atributos que son flexibles. [15] HTML5 se convierte en un conjunto de tencologías con JS y CSS. Varias etiquetas cambian para optimización de espacio y estilos.
  10. ¿Qué componentes de estructura y estilo tiene una tabla? Table: Define una tabla. Tr: Defina una fila en una tabla. Td: Define los datos en la tabla. Th: Define encabezado en la tabla. Caption: Define el título de tabla. Border (CSS): Define el borde. Border-collapse (CSS): Colapsa los bordes de la tabla. Colspan y Rowspan: Hacen que una columna o fila abarquen más de una. Text-align: Define la orientación del texto. Padding (CSS): Agrega espaciado. (w3schools, s.f.) [16]
  11. ¿Cuáles son los principales controles de una forma HTML5? accept-charset: Específica el charset usado en la forma enviada. action: Específica una dirección url donde subir la forma. autocomplete: Específica si el navegador debe de completar automáticamente la forma. method: Específica el método 'HTTP' usado al subir la forma. name: Específica el nombre para identificar a la forma. target: Especifica el objetivo de la acción. enctype: Especifica el encoding en la información a registrar. novalidate: Específica al navegador que no debe válidar la forma. (w3schools, s.f.) [17]
  12. ¿Qué tanto soporte HTML5 tiene el navegador que utilizas? 456 de 555 puntos, utilizando Safari 12.0.2 en macOS Mojave 10.14.
  13. ¿Cuál es el ciclo de vida de los sistemas de información? Según David Lorge, los sistemas de información (SI) envejecen al igual que la gente. No podemos prevenir el envejecimiento de estos, pero podemos entender sus causas, limitar sus efectos y revertir temporarmente el daño que estos causen. Las causas del 'software aging' pueden ser porque el SI no puede ponerse al corriente con las nuevas necesidades, falta de uso o modificaciones 'ignorantes'.[18]
  14. ¿Cuál es el ciclo de desarrollo de sistemas de información? Según TutorialsPoint, el ciclo de vida del desarrollo Software (SDLC en sus siglas inglesas), es una secuencia estructurada y bien definida de las etapas en Ingeniería de software para desarrollar el producto sofware deseado. Algunas de las etapas son: Comunicación, Requisitos del Sistema, Estudios de factibilidad, Diseño de Software, Códigos, Pruebas, Integración, Implementación, Operaciones y Mantenimiento y Disposición. Algunos de los modelos de desarrollo son Modelo en Espiral, Modelo V y Modelo Big Bang. [19]
