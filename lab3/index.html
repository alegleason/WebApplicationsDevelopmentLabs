Preguntas

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    Antes que nada hay que saber para qué es el !important. Lo que hace este elemento es reemplazar algún estilo que venga por defecto en algún framework para front-end como Bootstrap, UIKit, Materialize u otro, con alguno estilo que tú como ingeniero definas. Mi recomendación es usarlo siempre y cuando sepas exactamente qué es lo que estás haciendo y por qué, y claro, cuando este nuevo elemento vaya a complacer las necesidades del cliente.
  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    Esto es una de las cosas en las que más tiempo invierto (relativamente): elegir imágenes, íconos, tipografía y demás; pero sobre todo, es muy difícil elegir imágenes de fondo. Hay muchos aspectos que hay que cuidar para que el sitio se vea responsivo. Primero debemos elegir una imagen con buen tamaño (o configurarla para que tenga un buen tamaño) y no se vea pixeleada al cubrir todo el background; otra opción es elegir una imagen especial de CSS que permita que se vea continuidad (se ve como una sola imagen de fondo, pero en realidad son varias en cuadricula, ejemplo de esta es la actual); por último probar nuestra imagen en diversos tipos de tamaño de pantalla para comprobar una buena visualización.
  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    Las tres son unidades distintas y a fin de cuentas se utilizan para diversos propósitos, yo los clasificaría dela siguiente manera:
    • Porcentaje (%): Utilizarlo para definir el tamaño de containers, imágenes, tablas y cualquier bloque dentro de la página ya que son muy fáciles de visualizar mentalmente y son altamente responsivos -al menos en lo que a mí me ha tocado-. El porcentaje te permite divisar el tamaño del bloque que estés manejando con respecto a la página cómodamente.
    • Puntos (pt): Utilizarlo para definir tamaños de letras principalmente. La diferencia entre este y px es a veces sutil por lo que a veces no hay problema en intercambiar entre uno y otro. Cualquier cosa que tenga relleno puede ser definida con puntos, no espaciados.
    • Pixeles (px): Utilizarlo para padding (ya que esta medida no se puede definir en pt), para bordes y márgenes. De igual manera es una medida sencilla de estructurar mentalmente, y de jugar con ella.
  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    Básicamente lo que hace la versión minimizada es remover caracteres innecesarios, tales como espacios, tabulaciones o enters propios del CSS. Evidentemente es menos pesada que una versión no minimizada y esto es el factor que va a hacer que su carga sea más rápida. Es bueno usarlo cuando el archivo CSS ya es final o pertenece a algún framework como Boostrap, ya que al abrirlo su entendimiento no se logra tan fácil como en una versión sin minimizar (imagina todo un código en una sola linea sin ningún tipo de espaciado, comparado con otro estructurado regularmente).
